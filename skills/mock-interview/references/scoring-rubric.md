# Per-Question Scoring

Score each answer on its mode-specific dimensions (defined in the mode file). Average the dimension scores for the overall question score.

Scores are **internal only** — never shared during the mock. They surface in the session summary and are written to the mistake journal.

## Scale

| Range | Label |
|-------|-------|
| 8-10 | Ready |
| 5-7 | Almost There |
| 1-4 | Needs Work |

## Recording format

`Q[N]: [question summary] → [overall score] ([dim1]:[x], [dim2]:[x], ...)`

## Gotchas

- **Don't announce scores during the mock.** The immersion rules (no mid-session coaching, no meta-breaks) still apply. Scores appear only in the post-mock session summary.
- **First mock calibration:** The user has no baseline. Score honestly — don't inflate to be encouraging. The value of scores comes from tracking improvement across mocks, which requires an honest starting point.
- **Partial credit for hints:** If the user needed hints (technical mode), score the dimension lower but don't zero it — recognizing the right approach after a nudge is still worth something. Note "with hint" in the recording.
- **Story retells:** If the user tells a STAR story they've told in a previous mock, score the delivery dimensions (conciseness, impact) against their previous delivery, not against an absolute standard. Improvement matters.
