# Mode: Cover letter

User asks for a cover letter. Often chains naturally from resume tailoring.

**Context reuse:** If a job posting was already discussed in the same conversation, reuse it — don't ask again. If no posting was discussed, check `memory/interview-context.md` -> `## Job Posting History` for recent postings.

**Step 1: Identify the target**

If no posting is available:

> "Which role is this for? Share the posting if you have it, or just the company and role title."

**Step 2: Draft the cover letter**

Structure:
1. **Hook** — why this company specifically (not generic "I'm excited about your mission"). Reference something concrete about the company, their product, their team, or a recent development.
2. **Relevant experience** — 2-3 strongest matches between the user's background and the role requirements. Pull from resume context and win log.
3. **What makes the user rare** — the unique combination of skills, background, or perspective that other candidates probably don't have.
4. **Close** — forward-looking, confident without being presumptuous.

**Tone:** Match to the company. Read cues from the job posting:
- Startup / casual posting -> conversational, show personality
- Enterprise / formal posting -> professional but still human
- Creative industry -> more expressive
- When in doubt, lean slightly conversational — it's more memorable

**Step 3: Iterate**

Present the draft and invite feedback:

> "Here's a first draft. The hook focuses on [X] — if there's something else that draws you to [company], let me know and I'll rework it."

Revise based on user feedback. Multiple rounds are normal for cover letters.

**Rules:**
- Never write a generic cover letter — every sentence should be specific to this company + this user
- Don't open with "I am writing to express my interest in..." or similar cliches
- Keep it under 400 words unless the user asks for more
- Reference real projects, wins, and experiences from memory — not vague claims
