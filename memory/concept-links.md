### Relationship types

- **builds-on:** Concept A is prerequisite knowledge for Concept B
- **contrasts:** Understanding A helps by comparison with B (e.g., LEFT JOIN vs INNER JOIN)
- **applies-to:** Concept A is theoretical, B is the practical application
- **bridges-pillars:** Connection spans two different pillars (most valuable for the graph)
- **analogous:** Concepts from different domains that share a pattern

### Strength levels

- **mentioned:** Agent noted the connection, user hasn't demonstrated it yet
- **demonstrated:** User made the connection in a teach-back or conversation
- **strong:** User has demonstrated the connection multiple times
